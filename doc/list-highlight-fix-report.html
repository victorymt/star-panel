🗺️ star-panel 列表高亮修复报告

项目： Starcatch star-panel（Quickshell QML 负一屏组件）
现象： vim/emacs 键盘绑定已就绪（j/k/gg/G/h/l/o/dd），但上下移动时没有视觉高亮反馈，用户不知道当前导航到哪一项。

📅 时间线

第 1 轮
增加 vim/emacs 绑定（j/k/h/l/o/gg/G/dd/Ctrl+N/P/A/E）
绑定后测试发现 ListView.isCurrentItem 没有生效，高亮不可见。认为是对比度不够。
第 2 轮
强化视觉参数（弯路 1）
修改 background 颜色从 surface1 0.4 alpha → surface2 0.5， 增加 2px 蓝色边框，添加 ▸ 指示符。
结果：还是看不见。误以为颜色仍然不够深，继续调参。
第 2.5 轮
改 currentIndex 初始值（弯路 2）
把 currentIndex: -1 改成 0，认为首项不高亮是因为初始索引为 -1。
结果：仍然看不到任何高亮。
误解：绑定本身是好的，只是初始值不对。
第 3 轮
用 explore agent 深入调研（根因）
ListView.isCurrentItem 是 attached property，在 delegate 内嵌套子项 （如 background: Rectangle）中引用时，作用域是 Rectangle 自身 而非 delegate root，永远拿不到 true。
再次阅读 Qt 自带 5 个 ItemDelegate 样式（Basic、Fusion、Material、Universal、FluentWinUI3）， 确认它们全部用 control.highlighted 而非 ListView.isCurrentItem。
第 4 轮
最终修复（正确）
delegate root 绑定 highlighted: ListView.isCurrentItem， background 和 contentItem 中通过 itemDel.highlighted 引用。三层视觉标识立即可见。

🧭 阶段一（弯路）：误判为对比度问题

❌ 错误诊断

分析 TodoList.qml 中 delegate 的 background 代码，发现背景色是 surface1 0.4 alpha。认为看不清楚是因为颜色太浅： 

// ❌ 弯路：调颜色参数
background: Rectangle {
    color: ListView.isCurrentItem                              // ← 永远 false！
        ? Qt.rgba(colors.surface2.r, colors.surface2.g, colors.surface2.b, 0.5)
        : "transparent"
    border.width: ListView.isCurrentItem ? 2 : 0              // ← 永远 0
    border.color: colors.blue
}
误解 1：以为 ListView.isCurrentItem 在任何层级都能正确解析为 true。 误解 2：以为问题只是颜色不够深、缺少边框和指示符、或者初始 currentIndex 是 -1。 这三个"修复"全都建立在错误的前提上（绑定能拿到 true），所以全部无效。

❌ 同时做的"修复"

  • currentIndex: -1 → 0（首项初始不高亮？）
  • 添加 ▸ 指示符 Text，visible: ListView.isCurrentItem → 永远 hidden
  • 添加 border，宽度 2，蓝色 —— 绑定读不到 true，边框为 0

效果：
所有键盘绑定都能正确移动 currentIndex（用 print() 印证）， 但视觉上完全看不出来 —— 画布上没有任何一个像素随 currentIndex 变化而变化。

🔬 阶段二（根因）：QML attached property 作用域

根本原因

ListView.isCurrentItem 是 attached property（附加属性）。 在 QML 中，attached property 的作用域绑定到声明该属性的对象自身。

当 delegate 结构是：

ItemDelegate {                          // ← ListView.isCurrentItem 附加到这里
    contentItem: RowLayout {
        // ... ListView.isCurrentItem 引用 RowLayout 自己（没有该属性）→ false
    }

    background: Rectangle {            // ← ListView.isCurrentItem 引用 Rectangle 自己
        color: ListView.isCurrentItem   //    （没有该属性）→ 永远 false
            ? someColor : "transparent"
    }
}

background 内部的 ListView.isCurrentItem 解析的是 Rectangle 实例 而非 ItemDelegate 实例。 Rectangle 上没有 isCurrentItem 这个 attached property，所以永远返回 false。

💡 关键发现

Qt 内置的 5 个 ItemDelegate 样式（Basic、Fusion、Material、Universal、FluentWinUI3） 没有一个在 background 或 contentItem 里直接使用 ListView.isCurrentItem。 它们全部在 delegate root 绑定 highlighted: ListView.isCurrentItem， 然后子项引用 control.highlighted（普通属性）。

这是 Qt 框架自己的最佳实践 —— 我们的代码在犯一个 Qt 文档有明确示例避免的错误。

✅ 阶段三（最终修复）：三层视觉 + 正确绑定

修复方案

核心改动：把 ListView.isCurrentItem 的解析提升到 delegate root，子项通过 root 的 id 引用普通属性。

// ✅ 正确写法
delegate: ItemDelegate {
    id: itemDel
    highlighted: ListView.isCurrentItem     // ← 在 root 正确解析 attached property

    contentItem: RowLayout {
        Text {
            text: "▸"
            visible: itemDel.highlighted    // ← 通过 id 引用普通属性
        }
    }

    background: Rectangle {
        color: itemDel.highlighted          // ← 通过 id 引用普通属性
            ? Qt.rgba(colors.surface2.r, colors.surface2.g, colors.surface2.b, 0.5)
            : hovered
                ? Qt.rgba(colors.surface1.r, colors.surface1.g, colors.surface1.b, 0.3)
                : "transparent"
        border.width: itemDel.highlighted ? 2 : 0
        border.color: Qt.rgba(colors.blue.r, colors.blue.g, colors.blue.b, 0.8)
    }

    onClicked: {
        detailPopup.type = "todo";
        detailPopup.itemData = modelData;
        detailPopup.open();
    }
}

层级效果代码

底色 surface2 0.5 alpha 半透明底色 color: itemDel.highlighted ? rgba(surface2, 0.5) : ...

🧠 教训总结

  1. 先验证绑定值是否正确，再调样式参数。
    在 background 里 print(ListView.isCurrentItem) 能立即暴露问题 —— 但它 永远输出 false。如果第 2 轮就验证这一点，可以跳过弯路直接到第 4 轮。
  2. QML attached property 有严格作用域。
    ListView.isCurrentItem、GridLayout.isRow 等附加属性只附加到声明的父组件， 子组件拿不到。必须在 root 读一次 highlighted: ListView.isCurrentItem， 子组件通过 rootId.highlighted 引用。
  3. ItemDelegate 内置了 highlighted 属性。
    这是 Qt 官方和所有内置样式推荐的高亮绑定方式。直接使用 highlighted: ListView.isCurrentItem 然后引用 itemDel.highlighted 符合框架设计，避免了 attached property 的作用域问题。
  4. 不要在一个错误假设上调参三次。
    颜色从 surface1 0.4 → surface2 0.5、加 border、加 ▸、改 currentIndex —— 这四个改动全部基于 "绑定已经能读到 true" 的错误前提。任何一次停下来验证 print() 输出都能省下两轮弯路。
